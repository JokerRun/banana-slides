Banana Slides · From Description 图片上下文修复方案

问题不是“图片没传”，而是“图片语义位置被切断”。

我建议不要只删除 remove_markdown_images()。如果要做，就一步到位：保留二进制参考图，同时把 markdown 图片转换为稳定、结构化、provider-friendly 的 IMAGE_REF 语义标记和 manifest，让模型同时知道“看哪张图”以及“这张图该服务哪段内容”。

根因：backend/services/ai_service.py:426-428 当前：binary refs ✅ / prompt context ❌ 推荐：structured image refs + binary refs

推荐方案：一步到位 Hybrid，不走“先删调用试试看”

1. 保留视觉输入

继续通过 additional_ref_images 传图片二进制，确保 provider 真能“看到”图。

2. 保留语义绑定

把 ![alt](/files/x) 转成 [IMAGE_REF:image_1]，留在页面文案位置。

3. 显式给模型 manifest

在 prompt 中追加“图片引用清单”：id、caption、source、使用规则、放置意图。

为什么不用原始 markdown 直接透传？
  • 不同 provider 对 markdown URL 的理解不稳定。
  • /files/... 对外部模型未必可访问。
  • 结构化 marker 可避免模型把 URL 当作可访问资源，而是把它当作已附加 reference image 的索引。
为什么仍保留 binary refs？
  • 模型真正看图仍依赖 binary/reference image。
  • manifest 只负责“这张图是什么、和哪段内容绑定、怎么用”。
  • 两个通道职责清晰，不互相替代。

风险与我建议的决策

Provider 是否理解 marker？

marker 是 prompt 语义，不要求 provider 访问 URL；风险低于原始 markdown URL。

是否会重复画图？

Prompt 要明确：“使用附加参考图，不要重绘成无关素材；必要时保持主体”。

直接实施 Hybrid。不要只删 remove_markdown_images；那是半修复，provider 行为不够可控。
