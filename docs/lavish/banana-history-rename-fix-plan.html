历史项目重命名失败，不是 API 没写入，是“标题读取源”和“写入字段”不一致

当前历史页保存重命名时写入 Project.idea_prompt；但卡片标题对非 restyle 项目优先读取第一页 Page.outline_content.title。所以生成 PPT / 翻译 PPT 改名后会被旧的第一页标题覆盖；restyle 特判优先读 idea_prompt，因此看起来正常。

Recommended Fix

新增显式项目名字段

不要继续复用 idea_prompt。项目显示名应该是独立 metadata：Project.project_name。

影响范围：Project model + migration + PUT API + frontend title utility + history rename tests。

用户现象
生成 PPT / 翻译 PPT 重命名不生效
为什么 restyle 正常
creation_type === 'restyle' 特判读 idea_prompt
核心风险
idea_prompt 被当作项目名，语义已经过载

1. 当前数据结构：Project 没有“项目显示名”

字段当前语义被重命名功能如何使用问题
Project.idea_promptidea 类型原始用户 prompt；restyle/translate 类型源文件名历史页把新名称写进这里语义过载 不是所有类型都会拿它当标题显示
Project.description_textFrom Description 的原始页面描述不参与重命名不能改，否则会破坏生成输入
Page.outline_content.title页面标题 / Slide 标题非 restyle 历史卡片优先显示它覆盖重命名结果，且不应该把项目改名写到第一页标题
Project.project_name不存在应该成为重命名的唯一写入目标建议新增

2. 失败链路：写入成功，但下一次渲染读错字段

flowchart LR
  A["History card edit"] --> B["PUT /api/projects/:id\n{ idea_prompt: newTitle }"]
  B --> C["DB Project.idea_prompt updated"]
  C --> D["setProjects updates idea_prompt"]
  D --> E["ProjectCard renders getProjectTitle(project)"]
  E --> F{"creation_type === restyle?"}
  F -- yes --> G["return project.idea_prompt\n✅ renamed title"]
  F -- no --> H["return first page outline title\n❌ old title wins"]

4. 推荐方案：新增 Project.project_name，重命名只改它

原则：项目名是项目 metadata；第一页标题是幻灯片内容；idea/description/source file 是生成输入。三者不要互相覆盖。

Before

// Rename writes generation input
PUT /api/projects/:id
{
  idea_prompt: "客户想看的新项目名"
}

// Render ignores it for non-restyle
if project.creation_type === "restyle":
  return project.idea_prompt
else:
  return project.pages[0].outline_content.title

After

// Rename writes display metadata only
PUT /api/projects/:id
{
  project_name: "客户想看的新项目名"
}

// Render always respects explicit project name first
if project.project_name?.trim():
  return project.project_name
else:
  return deriveDefaultTitle(project)

function getProjectTitle(project):
  if trim(project.project_name):
    return project.project_name

  if project.creation_type in ["restyle", "translate"] and project.idea_prompt:
    return project.idea_prompt       // source filename fallback

  if firstPageTitle(project):
    return firstPageTitle(project)   // old behavior fallback

  return project.idea_prompt || "未命名项目"

7. 测试建议

用例准备数据断言
生成 PPT 项目重命名creation_type='descriptions'，第一页 title = “第一页标题”PUT project_name='新名称' 后，History 显示“新名称”，第一页标题不变
翻译 PPT 项目重命名creation_type='translate'，idea_prompt='source-file'，第一页 title = “Slide 1”显示 project_name；清空/不存在时 fallback 到 source filename
Restyle 兼容creation_type='restyle'，idea_prompt='deck-a'未重命名前仍显示 deck-a；重命名后显示 project_name
API 校验project_name=' '返回 400；不写入空标题

8. 不推荐的快速修法

方案为什么看似简单为什么不推荐
让 getProjectTitle() 对所有类型都优先读 idea_prompt只改前端一处会改变现有 idea/description 项目的默认命名行为；继续污染 idea_prompt 语义
重命名时同步改第一页 outline_content.title立即让卡片显示新名称会修改 PPT 内容本身；用户只是改项目名，不是改第一页标题
只针对 translate/descriptions 做 if/else改动小继续堆类型特判，下一种 workflow 还会再破
